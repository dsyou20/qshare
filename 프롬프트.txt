사용자는 자신의 QGIS Python 스크립트를 관리하고 공유할 수 있는 웹 애플리케이션을 원합니다. 스크립트는 개인적으로 저장하거나 특정 사용자 또는 전체 공개로 공유할 수 있어야 합니다. 공유된 스크립트는 다른 사용자가 삭제할 수 없으며, 수정하려면 별도의 복사본을 생성해야 합니다.

핵심 기능 요구사항
1️⃣ 사용자 인증

로그인 (JWT 기반)
회원가입 (비밀번호 해싱)
사용자 정보 조회 및 수정
2️⃣ 스크립트 관리 (내 스크립트)

스크립트 생성, 수정
특정 사용자 또는 전체 공개로 공유
공유된 스크립트는 삭제 불가 (공유 취소 가능)
스크립트 사용 빈도 데이터 제공
3️⃣ 공유된 스크립트 관리

내가 공유한 스크립트 목록 조회
공유 취소 기능 제공 (삭제 불가)
공유된 스크립트 사용 예시 자동 생성
4️⃣ 찜한 스크립트 (즐겨찾기 기능)

공유된 스크립트 찜하기
찜한 스크립트 목록 조회
공유된 스크립트를 복사하여 내 스크립트로 저장 (새 이름으로 저장)
💡 특히, QGIS 플러그인과 연동하여 로그인 후 내 작업(스크립트) 동기화 및 등록, 수정이 가능해야 함.

📌 API 문서화 및 개발 도구

Swagger(OpenAPI) 기반 API 문서 제공
Prisma ORM을 활용한 DB 스키마 관리
DTO를 활용한 요청 및 응답 데이터 구조화
위 요구사항을 기반으로 웹 애플리케이션을 개발할 수 있도록 구성
